01 — About

What is GitHub?

GitHub is a web-based platform that provides hosting for software development projects using Git — enabling version control, collaboration, and sharing of code among developers.

Git keeps track of every change in your code, so you can view the full history, undo mistakes, and manage different versions of a project simultaneously.

In short, GitHub is a platform for version control and collaboration that lets developers store, share, and work on code together — from solo projects to global open-source communities.

02 — Interface

The GitHub Interface

GitHub's interface is organized into dashboards, repositories, and tabs that cover code, collaboration, and project management.

🔍

Navigation Bar

Top bar with links to search, notifications, your profile, and global settings.

🔎

Search Bar

Search for any repository, user, or code snippet across GitHub's entire platform.

📁

Repository List

Displays your own repositories and any repos you've contributed to recently.

📰

Activity Feed

Live updates on commits, pull requests, and issues from projects you follow.

💻

Code Tab

Displays files and folders. Browse code, read docs, or clone/download the project.

🐛

Issues Tab

Track bugs, tasks, and feature requests. The central hub for project discussion.

🔀

Pull Requests

Where code changes are proposed, reviewed, discussed, and merged into the main branch.

⚙️

Actions Tab

Automated CI/CD workflows — like running tests or deploying on every push.

📋

Projects Tab

Kanban-style boards for organizing tasks, milestones, and project management.

📖

Wiki Tab

Built-in documentation and guides written by the repo maintainers.

🛡️

Security Tab

Vulnerability alerts, dependency scanning, and security advisory reports.

📊

Insights Tab

Analytics on repo activity, contributor stats, traffic, and code frequency.

📄

File Contents

Displays the raw content of any file with syntax highlighting for supported languages.

✏️

Edit files directly in the browser (if you have write permission), view revision history, and see who changed what line with git blame.

Shows your personal bio, pinned repositories you want to highlight, and links to your social or personal site.

03 — Repositories

How to Create a Repository

After logging into your GitHub account, follow these steps to create your first repository.

01

Click the + icon

Find the plus icon in the top-right corner of the GitHub navbar, then select New repository from the dropdown.

02

Enter a repository name

Choose a short, memorable name for your project. For example: my-first-project. No spaces — use hyphens.

03

Add an optional description

Write a brief summary of what your project does. This appears on your profile and in search results.

04

Choose visibility

Select Public (visible to everyone) or Private (only visible to you and collaborators).

05

Click Create repository

That's it! Your repository is live. GitHub will guide you through pushing your first code.

04 — VS Code Integration

Linking VS Code with GitHub

Connecting your local VS Code editor to GitHub allows you to push and pull code without leaving your editor.

Step 1

Install Git

Download and install Git from git-scm.com. Git is the underlying version control system that GitHub is built on. Verify it works by running git --version in your terminal.

Step 2

Configure Your Identity

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Only needs to be done once per machine.

Step 3

Open Your Project in VS Code

Open any project folder in VS Code. Use the built-in Source Control panel (the branch icon in the sidebar) to initialise Git and connect to your GitHub remote.

Step 4

Authenticate with GitHub

The first time you push to GitHub, VS Code will prompt you to sign in. Follow the browser-based OAuth flow — once authenticated, your credentials are saved securely.

05 — Collaboration

How to Collaborate on GitHub

GitHub makes it simple to invite others to work on your project.

  1. 1
    Open your repository

    Navigate to the repository you want to share on GitHub.

  2. 2
    Go to Settings → Collaborators

    In the repository's top navigation, click Settings, then select Collaborators from the left sidebar.

  3. 3
    Add a collaborator

    Click Add collaborator and search by their GitHub username or email address.

  4. 4
    They accept the invite

    Your collaborator receives an email invitation. Once they accept, you can both push and pull changes freely.

💡
Pro Tip: Use Pull Requests

Even when collaborating directly, use Pull Requests to propose changes. It keeps a clear record of what changed and why, and allows review before merging into the main branch.
